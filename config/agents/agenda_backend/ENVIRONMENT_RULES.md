# agenda_backend Environment Rules

- Ambienti: `local`, `demo`, `staging`, `production`.
- Non usare URL di produzione in demo.
- Demo usa endpoint `agenda_core` demo separati.
- Non committare segreti o token reali.
- Per documentazione completa: `config/docs/agenda_backend-environments.md`.
- Per demo: `config/docs/agenda_backend-demo-environment.md`.
